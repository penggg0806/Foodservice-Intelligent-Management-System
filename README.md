餐饮门店智慧供应链与库存智能决策系统
本项目是一款面向餐饮零售门店的精细化供应链与智慧库存管理系统。系统基于高度解耦的模块化架构（MVC 思想）开发，将关系型数据仓储、Agnes-2.0-Flash 多模态大模型智能网关、自适应时序预测及智能补货行动指南深度集成，彻底解决餐饮门店传统库存管理中盘点耗时、补货全凭经验、前台收银与底层食材脱节、损耗难以控制的实际痛点。

🌟 核心技术亮点与架构优势
统一 AI 智能数据网关（大一统设计）
系统打破了传统进销存软件各模块孤立录入的壁垒。无论是店长的口语化记账文本、前台收银台传来的下单口令，还是 Excel 初始库存表/纸质盘点单的拍照截图，全部交由 Agnes-2.0-Flash 统一作为数据入口进行多模态全表穿透分析，自动清洗为标准结构化事务列表。

支持批量事务与自适应建档（Excel 一键初始化）
AI 网关原生支持批量列表（List）输出。当直接上传包含几十种食材的期初盘点表截图时，系统会自动在底层进行高并发事务循环，若发现新食材会自动在资产库中建档，实现“一张截图搞定初始库存建立”。

复合套餐与单品 BOM 双层穿透扣料
前台收银模拟结账时，系统可无缝识别单品或复合套餐。在数据库事务（Transaction）锁的控制下，一秒内将套餐自动拆解为单品、单品依据标准配方（BOM）秒级穿透扣减底层多项对应原材料的当前库存，实现了“物理库存 = 历史入库 - 销售扣料 - 打烊报损”的精准全链路闭环。

安全合规隔离机制
彻底隔离 API Key，采用 .env 环境配置文件与系统环境变量加载密钥，防止敏感信息泄露。

企业级可观测性与容错链
在 AI 流水账抓取层引入 Pydantic 强校验链，保障核心事务不崩溃；全面移除了隐式吞错，深度接入标准 logging 模块，所有底层 SQL 事务或网络请求异常均有据可查。

💾 数据库方案设计 (SQLite)
系统底层设计并关联了 7 张核心业务表，以保障全链路的数据一致性：

items_config：食材/商品基础配置表（维护当前库存、安全库存、特征损耗率及单位）。

dish_bom：单品物料清单配比表（存储单品菜品与底层基础原材料的消耗配方）。

combo_config：复合套餐组成配置表（存储前台套餐与多项单品菜品的组合关系）。

sales_history：销售与外部环境历史表（记录每日各原材料扣料消耗总量、气温、废弃量，供时序模型分析）。

purchase_orders：采购订单记录表（审计每次采购的数量、总成本及供应商）。

inventory_transactions：库存流水变动审计明细表（记录所有采购入库、销售扣料、打烊报损等每一次波动的明细原因，供全程溯源）。

ai_reports：AI 报告缓存表（缓存当日各食材的智能备货指南，避免重复调用大模型）。

🚀 环境安装与快速上手
1. 克隆或创建项目
确保您的电脑已安装 Python 3.9 或更高版本。将源码文件放置在对应目录中，并在项目根目录下打开终端。

2. 安装扩展依赖
执行以下命令，一键安装系统所需的全部第三方扩展库：

Bash
pip install -r requirements.txt
3. 配置本地环境变量
在项目根目录下创建一个名为 .env 的文件，并填入您的 AI 接口密钥、接口地址及您锁定的模型型号：

代码段（本demo默认使用agnes大模型）
OPENAI_API_KEY=您的真实_sk_openai_api_key
OPENAI_BASE_URL=https://apihub.agnes-ai.com/v1
AI_MODEL=Agnes-2.0-Flash
4. 启动系统
在终端中运行以下命令，系统会自动激活并在您的浏览器中打开智能看板：

Bash
streamlit run app.py
注：系统首次运行时，会自动激活自适应修复与种子注入层，自动建立所需的全部 7 张表结构并注入 21 天的历史销售数据、标准单品配方（宫保鸡丁、黑椒牛柳）及复合套餐（超值双人套餐等），方便您无缝进行功能测试。

⚙️ 业务功能模块简介
AI 智能流水录入中枢网关：支持日常口语化输入（如：进货、报损、点单文本）或上传收据、Excel 初始库存表截图，交由 Agnes-2.0-Flash 进行高精度全表穿透分析，秒级完成批量入库与自适应建档。

前台收银模拟器：模拟前台快捷结账。下单时自动将订单合成口令交由 AI 网关处理，自动探查是否为套餐并根据标准 BOM 配方秒级更新底层原材料库存，同时累加至时序销售历史中。

实时库存盘点与断货预警：提供直观的库存水位线监测。当现有库存低于系统设定的安全库存时，指标卡自动触发红色高亮预警，动态提醒店长补货。

销量预测分析：提取历史销量并结合气温等外部协变量进行统计学多项式拟合，精准预测未来 3 天的商品销量走势，并提供局部回测误差（MAE）及 95% 动态可信区间图谱。

智能补货行动指南：AI 决策智能体根据用户选择的抗风险策略（优先防断货或优先控成本）动态调整备货线，将复杂的数学模型输出翻译为店长“大白话”的备货、安全库存核对行动建议。

商户资产配置大本营 (CRUD)：提供食材/商品的资产档案建立与淘汰注销控制功能，为供应链和 BOM 配方提供元数据支撑。
